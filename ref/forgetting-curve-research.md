# 艾宾浩斯遗忘曲线融入分析报告

> 调研日期：2026-06-04
> 核心文献：Hermann Ebbinghaus (1885) 《记忆：关于实验心理学的贡献》

---

## 一、遗忘曲线要点

### 1.1 核心发现

Ebbinghaus 在 1885 年的实验中发现（使用无意义音节）：

| 时间 | 保留率 |
|------|--------|
| 刚学完 | 100% |
| 20 分钟后 | ~58% |
| 1 小时后 | ~44% |
| 9 小时后 | ~36% |
| 1 天后 | ~34% |
| 2 天后 | ~28% |
| 6 天后 | ~25% |
| 31 天后 | ~21% |

**关键洞察**：遗忘不是匀速的——最初几小时忘得最快，之后逐渐平缓。每次有效回忆会重置遗忘曲线，且遗忘速度显著减慢。

### 1.2 间隔复习的最优节奏

基于后续研究（Pimsleur, 1967; Cepeda et al., 2006），有效复习节奏为：

| 复习次数 | 间隔 | 说明 |
|---------|------|------|
| 第 1 次 | 学完后 1 天 | 对抗最大跌幅 |
| 第 2 次 | 3 天后 | 强化记忆 |
| 第 3 次 | 1 周后 | 转入长期记忆 |
| 第 4 次 | 2 周后 | 巩固 |
| 第 5 次 | 1 个月后 | 几乎永久保留 |

## 三、实施建议

### 3.1 学习状态新增字段

在 `learning_state.py` 的 `progress.completed_units[]` 中，每个单元增加：

```json
{
  "name": "模块1：命名≠理解",
  "result": "pass",
  "completed_at": "2026-06-04T10:00:00",
  "next_review_at": "2026-06-05T10:00:00",     // ★ 新增：下次该复习的时间
  "review_count": 0,                              // ★ 新增：已复习次数
  "review_history": [                              // ★ 新增：复习记录
    // {"date": "...", "result": "pass"}
  ]
}
```

同时在顶层增加：
```json
{
  "review": {
    "last_check_at": null,    // 上次检查复习的时间
    "next_check_at": null     // 下次应该检查的时间
  }
}
```

### 3.2 复习调度算法

```
完成学习 → 设置 next_review_at = now + 24h

每次回来学习前：
  检查 progress.completed_units[].next_review_at
  找出所有 next_review_at < now 的单元
  如果有到期复习 → 先复习再学新内容
  复习通过 → next_review_at 按间隔递增
  复习不通过 → 重新教学（该单元）
```

间隔递增规则：
```
第 1 次复习通过 → next_review_at = now + 3 天
第 2 次复习通过 → next_review_at = now + 7 天
第 3 次复习通过 → next_review_at = now + 14 天
第 4 次复习通过 → next_review_at = now + 30 天
第 5 次复习通过 → 标记为"已巩固"，不再自动安排复习
```

### 3.3 SOP 改动

在 `ref/teaching-sop.md` 的 Phase 1（学情诊断）之前、用户回来时，增加**复习检查**步骤：

```
用户回来继续学习
    │
    ├─ 加载学习状态
    ├─ 检查 review.next_check_at < now？
    │   ├─ 否 → 直接进 Phase 1
    │   └─ 是 → 列出到期复习项
    │          └─ 用户选择先复习还是先学新的
    │
    └─ 复习流程（走快速回顾版本）
```

### 3.4 快速回顾 vs 完整教学的区分

复习不是重新教学——用"快速回顾"：

| 维度 | 初次教学 | 快速回顾 |
|------|---------|---------|
| 讲解 | 完整概念讲解 | 一句话唤醒："还记得 [概念] 吗？" |
| 类比 | 生活类比 | 用户自己回忆类比 |
| 练习 | 刻意练习循环 | 1 道检测题，通过就过，不通过回炉 |
| 时间 | 10-15 分钟 | 2-3 分钟 |

## 四、风险与边界

| 风险 | 级别 | 应对 |
|------|------|------|
| 用户觉得复习打断进度 | ★★☆ | 让用户选择"先复习还是先学新的"，不强制 |
| 复习间隔太死板 | ★☆☆ | 使用默认间隔但允许用户跳过 |
| 学了很久没回来，积压太多复习项 | ★★☆ | 最多复习前 3 个到期的，其余自动标记为"过期复习" |
| 用户从未回来 | ★☆☆ | 不主动推送提醒，只在下一次回来时检查 |
