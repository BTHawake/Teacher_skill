# 统一提取场景调研：女娲方法能否作为 teacher-skill 的通用管道？

> 调研日期：2026-06-04
> 问题：是否有必要把所有来源的资料（用户提供、网络搜索、开源 skill）都用女娲的方法统一提取思维模型，再进入教学流程？

---

## 一、现状：6 条路径，3 种提取方式

当前 teacher-skill 有 6 条路径（A-F），三种不同的提取方式：

| 来源 | 当前提取方式 | 提取质量 | 问题 |
|------|------------|---------|------|
| 用户资料 | AI 人工阅读 + 手动提取 | 取决于 AI 水平 | 没有标准框架，可能漏掉或偏颇 |
| 开源 skill | `reverse_distill.py` 机械分析 | 中——只提取了模型名称 | 没有深度验证，提取的是"说了什么"不是"怎么思考" |
| 网络搜索 | AI 手动搜索 + 手动提取 | 低——最依赖 AI 水平 | 没有系统化的搜索和分析框架 |

**核心问题**：三种来源用三种不同的提取方式，质量参差不齐，无法保证教学内容的底线。

---

## 二、女娲的提取方法论：值得统一吸收的部分

女娲.skill 的核心不是 6 个 Agent 并行，而是**一套验证过的提取方法论**：

### 2.1 心智模型三重验证（建议统一采用）

无论来源是什么（用户资料/skill/网络搜索），提取出的每个"可教知识点"都应该通过**三重验证**：

1. **跨域复现**——这个知识点在 ≥2 个不同场景中出现过？
   - 用户资料中：作者在不同章节中反复提到
   - 网络搜索中：多个独立来源提到同一个观点
   - 开源 skill 中：此人多次在不同话题中使用同一个思维模型
2. **生成力**——这个知识点能用来推断新问题？
3. **排他性**——这个知识点是此人独特的，还是常识？

### 2.2 提取维度裁剪（从 6 维减到 4 维）

女娲用 6 个维度提取，但对教学来说，4 个就够了：

| 女娲的 6 维 | teacher-skill 的 4 维 | 为什么保留 |
|------------|---------------------|-----------|
| 著作（系统性思考） | ✅ 保留为**核心观点** | 教学最重要的素材 |
| 对话（即兴思维） | ✅ 保留为**实际应用** | 展示思维在真实对话中的运作 |
| 表达（风格特征） | ❌ 合并到核心观点 | 教学不教表达风格，教思维方式 |
| 他者（外部评价） | ✅ 保留为**多视角** | 防止单一视角的盲区 |
| 决策（真实行为） | ✅ 保留为**案例** | 真实决策是最好的教学案例 |
| 时间线 | ❌ 不需要 | 教学不需要完整时间线 |

### 2.3 诚实边界（必须保留）

女娲要求每个产出必须有明确的"做不到什么"。teacher-skill 已经将其加到模板中，需要在统一管道中强制执行。

## 三、统一管道设计：单一入口、标准提取、统一输出

```
用户输入（任意形式）
    │
    ▼
┌─────────────────────────────────────────────┐
│             Step 1：来源归一化                 │
│                                              │
│  用户资料 ──→ 文本化（无论 PDF/链接/文档）      │
│  开源 skill ──→ 解析 frontmatter + 心智模型     │
│  网络搜索 ──→ 搜索结果摘要                      │
│  混合输入 ──→ 合并为统一文本池                  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│         Step 2：四维提取（统一框架）           │
│                                              │
│  ① 核心观点    从所有来源提取此人反复强调的      │
│                 核心理念（≥2次出现才采纳）       │
│  ② 实际应用    从对话/案例提取思维的"怎么用"     │
│  ③ 多视角      从他者评价/批评提取补充视角      │
│  ④ 案例库      从决策和事件提取教学案例         │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│      Step 3：三重验证（质量门）                │
│                                              │
│  每个候选知识点 → 跨域复现？→ 有生成力？→       │
│  → 有排他性？→ 通过 → 进入教学模块              │
│              → 未通过 → 降级为补充材料           │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│      Step 4：课程组装                         │
│                                              │
│  通过的知识点 → 按关联度分组 → 排序为模块       │
│  补充材料 → 作为模块内的案例/拓展阅读            │
│  诚实边界 → 根据来源质量和验证结果自动生成        │
│  来源标注 → 每个模块附上对应的来源链接           │
└─────────────────────────────────────────────┘
    │
    ▼
         进入 teacher-skill 教学流程（Phase 1→2→3→4→5）
```

## 四、改动成本评估

| 改动 | 影响范围 | 工作量 | 价值 |
|------|---------|-------|------|
| 四维提取框架文档化 | 新增 `ref/extraction-framework.md` | 小 | 高——所有路径统一标准 |
| 三重验证集成到 SOP | 修改 `ref/teaching-sop.md` Phase 0→1 之间 | 小 | 高——提升教学内容底线 |
| 来源归一化步骤 | 修改 `SKILL.md` 自动搜索流程 | 小 | 中——消除当前路径差异 |
| 统一管道替代 6 条路径 | `SKILL.md` 大幅重构 | 中 | 高——从 6 种逻辑简化为 1 种 |
| 女娲式 6 Agent 并行 | 不采用 | 0 | 不需要——我们是教学不是造人 |

---

## 五、结论

**有必要统一提取场景，但不采用女娲的 Agent 架构。**

应该吸收的是：

| 吸收 | 不吸收 |
|------|--------|
| ✅ 四维提取框架（核心观点/实际应用/多视角/案例库） | ❌ 6 个并行 Agent |
| ✅ 三重验证门（跨域复现/生成力/排他性） | ❌ 角色扮演规则/身份卡/表达DNA |
| ✅ 诚实边界强制执行 | ❌ 完整人物画像（颗粒度太细） |
| ✅ 来源标注规范 | |
| ✅ 统一管道替代 6 条独立路径 | |

**收益**：6 条路径 → 1 条管道。提取质量从"看 AI 水平"变为"有标准可依"。

**成本**：新增 `ref/extraction-framework.md`（~200 行文档），修改 SKILL.md 自动搜索段落。

要不要实施？
