# 技能教学操作指引

## 这是什么

AI 领域有一个爆发生态：**人格 Skill**——把某个人/角色的思维模式、表达方式、知识体系打包成 SKILL.md 文件。这些 Skill 包含丰富的可教知识。

技能教学做的：**把 Skill 中的可教知识提取出来，按照 teacher-skill 的教学流程，教会人类这些知识和思维方式。**

| | Skill 原本的用途 | 技能教学的用途 |
|--|----------------|----------------|
| 目标 | 让 AI 学会 X 的思维方式 | 让人类学会 X 的思维方式 |
| 产物 | SKILL.md（AI 指令） | subjects/xxx-teacher-SKILL.md（课程） |
| 使用者 | AI Agent | 人类学习者 |
| 可迁移性 | 绑定 AI 平台 | 你的大脑随时可用 |

> 用户不需要知道这个流程的存在。用户只需要说"我想学会巴菲特的思维方式"，
> AI 会自动完成搜索、分析、生成、教学的全流程。

## 触发流程

当用户说以下内容时，自动开始执行：

- "我想学会XXX的思维方式"
- "我想知道XXX是怎么想的"
- "帮我像XXX那样思考"
- "我想学XX的思维模式"
- "XX的思考方式是什么"

## 操作流程

### Step 1：识别对象

从用户的话中提取要学习的人物/角色名。

### Step 2：检查本地已有课程

查看 `subjects/` 目录下是否有对应的教学文件。如果有，直接进入 Phase 1 开始教学。

### Step 3：搜索开源 Skill 仓库

如果本地没有，到以下仓库搜索：

- https://github.com/tmstack/awesome-persona-skills（65+ skill）
- https://github.com/momozi1996/awesome-ai-persona-skills（100+ skill）

搜索时，访问仓库的 README 或 skill 目录，查找匹配对象。

### Step 4：分析 Skill

用 `scripts/reverse_distill.py --analyze <skill_url>` 分析 skill 内容，提取：
1. 基本信息：名称、类型（thinking/persona/writing）、行数
2. 心智模型：检测到的核心模型清单
3. 可教性评分

### Step 4B：技能来源不足时的网络搜索补充（v2.3）

如果找不到匹配的 skill，或 skill 内容不足以支撑完整课程时，自动转网络搜索：

1. **搜索方向**：
   - 搜索该人物的著作/演讲/访谈/文章
   - 搜索对该人物思维框架的分析文章
   - 搜索该人物的名言、核心观点、方法论
2. **提取可教内容**：
   - 核心思维方式（至少提取 3-5 个可教学的框架）
   - 与这些框架对应的生活例子
   - 该人物的独特视角和边界
3. **标注来源**：
   - 每个核心观点旁标注来源链接（`[来源](URL)` 格式）
   - 区分一手信息（本人原话）和二手信息（他人分析）
   - 在课程末尾附上完整的来源列表

### Step 5：判断是否适合教学

| Skill 类型 | 特征 | 适合教学？ | 策略 |
|-----------|------|-----------|------|
| **thinking（思维型）** | 有心智模型、决策启发式 | ✅ 非常适合 | 按心智模型拆分为教学模块 |
| **persona（角色型）** | 有身份卡、角色扮演规则 | ⚠️ 部分适合 | 提取背后的知识体系，不教角色扮演 |
| **writing（写作型）** | 有表达DNA、文风分析 | ✅ 适合 | 作为写作/表达训练课程 |

**不适合教学的 Skill**：纯情感陪伴型、纯角色扮演（无实质知识内容）。

### Step 6：生成课程文件

用 `--generate` 参数生成骨架文件：

```bash
python scripts/reverse_distill.py --analyze <url> --generate <subject-name>
```

然后在骨架上补充三个核心部分：

1. **概念讲解**：用大白话解释每个心智模型
2. **生活类比**：用日常生活中的例子让抽象概念可感知
3. **检测练习**：参考 `ref/verification-framework.md` 设计练习题

### Step 7：自动注册

将新生成的学科及其触发词添加到 `SKILL.md` 的学科引用表中。

### Step 7.5：双视角质量审查（v2.3 新增）

课程生成后、开始教学前，执行双视角审查。不要跳过——这是区分"能用"和"好用"的关键步骤。

**视角 A：教学效果审查**

以"学生"的视角通读整个课程文件，检查：

- [ ] 每个模块的概念讲解是否真的通俗易懂（不是术语堆砌）
- [ ] 每个类比是否贴切（不会误导）
- [ ] 检测练习是否能真正验证理解（不是"听了就懂"型）
- [ ] 分级教学是否覆盖弱/中/强三个层次
- [ ] 用户学完之后，能带走一个具体可用的东西吗？

**视角 B：内容准确性审查**

以"该领域专家"的视角通读，检查：

- [ ] 提取的心智模型是否忠于源 skill 的原意
- [ ] 有没有把 source skill 中的"角色扮演"部分错误地当成可教知识
- [ ] 每个模型的局限性是否写清楚了（不是只写优点）
- [ ] 诚实边界是否覆盖了足够的局限
- [ ] 触发词是否覆盖了用户可能说的各种方式

**处理冲突**：两个视角意见冲突时，以教学效果视角优先（内容准确但教不会 = 更差的结果）。

### Step 8：开始教学

完成课程生成后，自动进入 Phase 1 学情诊断，开始教学流程。

## 检查清单

每个新生成的课程完成后检查：

- [ ] 心智模型被完整提取
- [ ] 每个模型都有大白话的概念讲解
- [ ] 每个模型的生活类比
- [ ] 每个模型都有检测练习
- [ ] 有分级教学重点（弱/中/强）
- [ ] 每条核心观点有来源链接（网络搜索生成的课程必需）
- [ ] 已在 SKILL.md 中注册触发词
- [ ] 用户可以用"教我 [人名] 的思维方式"触发

## Skill 来源

| 源仓库 | 可用 skill 数 | 典型可教学 skill |
|--------|-------------|------------------|
| awesome-persona-skills (tmstack) | ~65 | 巴菲特、芒格、费曼、纳瓦尔、乔布斯 |
| awesome-ai-persona-skills (momozi1996) | 100+ | 24 位作家、13 位自媒体大V、天涯20大神 |
| 女娲.skill 生态 | ~50 | 各种思维型 skill |

## 边界

**适合教学的 Skill**：包含明确心智模型/思维框架的、包含系统知识体系的、包含可训练的表达方法的。

**不适合教学的 Skill**：纯情感陪伴型（目标是陪伴不是教学）、纯角色扮演型（无实质知识内容）、纯工具型（不是教学内容）。

# 增量更新模式（v2.3 新增）

当用户说"更新一下[人名]的课程"、"[人名]最近有新信息"、或提供了该人物的新素材时执行。

### 核心理念

不重写整个教学文件。只对变化的维度做增量追加，保留原有内容。

### 操作流程

**Step 1：读取现有课程**
加载 `subjects/<name>-teacher-SKILL.md`，识别当前状态：
- 心智模型有哪些
- 每个模型的"教学时间"标注
- 诚实边界中标注的局限

**Step 2：判断更新类型**

| 用户输入 | 更新类型 | 操作 |
|---------|---------|------|
| "[人名] 有新书/新观点" | 增量信息 | 在对应模块补充新案例，不修改已有内容 |
| "这个模型讲得不对" / "还缺一个重要的点" | 修正或补充 | 修正不准确的部分，或新增一个模块 |
| 用户提供了新的素材（文章/视频/文档） | 素材驱动更新 | 从素材中提取新信息，与现有内容融合 |
| "[人名] 的思想变了" | 版本更新 | 在诚实边界的"时效性"中标注变化，补充新立场 |

**Step 3：执行更新**

根据更新类型，执行对应操作：

- **增量信息**：在对应模块的"拓展阅读"或"案例"部分追加新内容，标注"（v2.3 补充）"
- **修正**：修改对应的讲解内容，在诚实边界中说明变更理由
- **新增模块**：在现有模块列表末尾追加新的模块编号
- **素材融合**：先分析新素材，提取与现有模型相关的信息，追加到对应位置

**Step 4：版本记录**

在文件的 frontmatter 中更新版本号（递增次版本号），并在文件末尾追加一条更新日志：

```markdown
---

## 更新日志

| 日期 | 版本 | 变更 |
|------|------|------|
| 2026-06-04 | v1.0 | 初始创建 |
| 2026-06-10 | v1.1 | 新增模块6：[模块名]；补充[人物]的最新观点 |
```

### 什么情况下应该重写而非增量更新

- 源 skill 已经完全重构（与原版差异 > 70%）
- 教学效果验证显示大部分用户无法理解当前的结构
- 发现原课程中提取的心智模型有根本性错误
