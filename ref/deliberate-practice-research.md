# 刻意练习 vs teacher-skill：融合分析报告

> 调研日期：2026-06-04
> 核心文献：Anders Ericsson《刻意练习》、K. Anders Ericsson (1993) "The role of deliberate practice in the acquisition of expert performance"

---

## 一、刻意练习的六大原则

### 1.1 原则速览

| # | 原则 | 核心含义 | teacher-skill 现状 |
|---|------|---------|-------------------|
| 1 | **目标明确** | 每次练习有具体、可衡量的目标，而非泛泛的"更努力" | ✓ Phase 2 制定学习路线时有目标设定，颗粒度不够细 |
| 2 | **专注投入** | 练习时必须全神贯注，不能三心二意 | ✓ Phase 3 要求逐概念讲解，已覆盖 |
| 3 | **即时反馈** | 每一次尝试后立即知道结果和对错 | △ 有验证回路但非即时（Phase 4 才验证，不是每一步都反馈） |
| 4 | **走出舒适区** | 练习的难度必须略高于当前能力水平 | △ 三级策略有分级但无"刻意超出"的设计 |
| 5 | **重复与修正** | 针对弱点的反复练习+修正，而非泛泛的做题 | ✗ 通过后就不再练习，没有"重复+变式"机制 |
| 6 | **心智表征** | 练习的目的是建立内部心智模型，而非机械记忆 | △ 跨学科模块有知识联网，但心智表征训练无专门设计 |

## 二、哪些可以直接吸收

### 2.1 HIGH PRIORITY：练习系统重构（改动中，价值高）

**当前问题**：teacher-skill 的练习是"一次性"的——用户做完练习题，通过就过，不通过就重新教学。没有"针对薄弱点的刻意反复"。

**建议改造**：引入"刻意练习循环"替代现有的"一次验证"。

当前：
```
讲解 → 练习题 → 通过→继续 / 不通过→重新教学
```

改造后：
```
讲解 → 练习题 → 
  ├─ 通过 → 变式练习（同一概念不同场景）→ 确认稳定理解 → 继续
  └─ 不通过 → 诊断具体薄弱点 → 针对性讲解 → 再次练习
         ↕ （最多循环 3 次，3 次还不过则降低难度级别）
```

**核心改动**：不是"做对了就过"，而是"做对了还要能在不同场景下做对"。

### 2.2 HIGH PRIORITY：反馈颗粒度细化（改动小，价值高）

**当前问题**：反馈只有"通过/不通过/部分通过"三种。用户不知道具体错在哪里。

**建议改造**：在验证框架中增加"反馈维度"：

```
当前：✗ 未通过
改造后：✗ 未通过
   ├─ 概念理解有偏差（具体：...）
   ├─ 场景迁移失败（能复述但不能应用）
   └─ 表达方式不准确（理解了对但说不清）
```

### 2.3 MEDIUM PRIORITY：心智表征训练（改动中，价值中）

**当前问题**：教学流程只教概念，不训练"在心里运行这个概念"的能力。

**建议改造**：在 Phase 3 中增加"心智演练"步骤：

- "不看资料，用自己的话在你的脑子里把这个过程过一遍"
- "如果[输入条件]变了，你的心智模型预测输出会怎么变？"
- "画一张图来代表这个概念（不用画出来，在心里画）"

### 2.4 MEDIUM PRIORITY：走出舒适区的"刻意"设计

**当前问题**：三级策略的练习难度是静态的——弱基础一直做选择题，强基础一直做应用题。但刻意练习要求**动态调整**难度。

**建议改造**：在三级策略基础上增加"难度微调"机制：

```
基础判定（Phase 1）：弱/中/强
    ↓
教学过程中动态调级（已有）
    ↓
新增：练习难度+1 原则
  - 用户连续 2 次通过当前难度 → 下一次题难度+1
  - 用户连续 2 次不通过 → 降回原来难度
  - 刻意保持"大部分时间在略微不舒服的难度"
```

## 三、不采纳的方面

### 3.1 刻意练习的"每天数小时高强度训练"模式

teacher-skill 面向的是日常学习场景，用户不可能像专业运动员那样每天练习数小时。我们不能要求用户"每天练 2 小时"。

### 3.2 刻意练习的"枯燥重复"设计

刻意练习的一个特征是"针对弱点的枯燥重复"——这适合专业训练，但不适合自发的学习场景。如果学习过程太枯燥，用户会流失。

**替代方案**：用"变式重复"替代"枯燥重复"——同一个知识点，用不同的场景反复练习，保持新鲜感。

### 3.3 对"天赋无用"的极端立场

Ericsson 认为天赋不重要，关键在练习时间和方法。但这个立场在学术界有争议（有证据表明某些能力的先天差异存在）。我们不采纳这种极端的立场。

## 四、实施建议

**改动位置**：`ref/teaching-sop.md` Phase 3 中的 Step 3.3（理解检查）+ `ref/teaching-strategies.md`

**具体改动**：

## 五、结论

| 维度 | 评估 |
|------|------|
| 方法论契合度 | ★★★★☆ 刻意练习的理念和 teacher-skill 的教学目标高度一致 |
| 当前差距 | ★★☆☆☆ 差距主要在"练习系统"和"反馈颗粒度" |
| 实施难度 | ★★☆☆☆ 3 个改动都是文档修改，不需要写新代码 |
| 价值 | ★★★★★ 从"教过"升级到"教到掌握"的关键一步 |

**建议**：实施 ≥2.4.1 版本。改造练习系统（从"一次验证"到"变式重复"）是最高优先级的改进。
